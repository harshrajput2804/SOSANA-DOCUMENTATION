Section 02

Project Profile

2.1 Project Definition

SOSANA Business Complaint, Review & Rating Platform is a multi-role, multi-tenant web application that functions as a Business Review, Rating, and Complaint Management Platform. integrate with the SOSANA blockchain ecosystem, enabling identity-verified consumer feedback on registered businesses.

Unique Selling Point: Unlike conventional review platforms, SOSANA Platform requires users to hold SOSANA tokens and complete on-chain KYC to file complaints — significantly reducing fake and spam submissions.

2.2 Project Description

The platform is a PHP/Laravel-based web application hosted on a standard LAMP/LEMP server or cloud infrastructure. It provides three distinct portals:

  • Public Portal: Browse businesses, read reviews, view complaint outcomes.
  • User Portal: Register, verify identity (KYC), write reviews, file & track complaints.
  • Business Owner Portal: Claim & manage business listings, respond to reviews and complaints, subscribe to plans.
  • Admin Panel: Full platform governance — manage businesses, users, reviews, complaints, subscriptions, payments, blog, ads, and system settings.

AI-powered features (AI Review Writer, AI Reviewer) are available as optional addons. The platform supports multiple payment gateways, multi-language, RTL, KYC verification, two-factor authentication, social login (Google, Facebook, Microsoft, VKontakte), and a comprehensive subscription/plan system.

2.4 Problem Statements

❌

P1 — Fake Reviews

Existing platforms are flooded with unverified, bot-generated, or purchased reviews distorting business trust scores.

❌

P2 — No Complaint Workflow

There is no standardised process for users to formally file, track, and resolve complaints against businesses.

❌

P3 — Lack of Identity Verification

Anonymous reviews and complaints cannot be trusted — there is no way to verify the reviewer's identity or stake.

❌

P4 — No Crypto-Scam Categories

Traditional review platforms have no specialised categories for rug pulls, exit scams, liquidity lock issues, or fake team socials.

❌

P5 — Business Unresponsiveness

Businesses rarely respond to consumer feedback. There is no mechanism to enforce response deadlines.

❌

P6 — Fragmented Admin Control

No centralised panel to moderate reviews, escalate complaints, manage subscriptions, and audit activity.

2.5 Need for New System

A purpose-built platform is needed that combines:

  • On-chain KYC & token-holding eligibility checks (via SOSANA dApp API)
  • Structured, multi-status complaint lifecycle management
  • AI-assisted review composition & spam detection
  • Business owner dashboard with response & subscription management
  • Comprehensive admin governance with audit logging
  • SEO-optimised public business directory with category/sub-category taxonomy
  • Multi-payment gateway integration (Stripe, PayPal, Crypto, 10+ gateways)
  • Multi-language, RTL, and theme customisation

Requirement Analysis

3.1 Feasibility Study

Technical Feasibility

The platform is built on Laravel 11 with PHP 8.2 — a mature, well-documented, and widely-supported technology stack. Standard LAMP/LEMP hosting is sufficient. All third-party integrations (SOSANA API, payment gateways, AI APIs) provide well-documented REST APIs. The technical feasibility is HIGH.

Operational Feasibility

The system is designed with intuitive role-based dashboards. Business owners and users require minimal training. Admin operations are self-explanatory. The platform includes built-in help documentation and email notifications for all lifecycle events. Operational feasibility is HIGH.

Economic Feasibility

Revenue is generated through tiered business subscription plans. Hosting costs are minimal (shared/VPS cloud). The base platform is a licensed SaaS product (one-time cost) with low ongoing maintenance. Economic feasibility is HIGH.

Schedule Feasibility

With the core platform already built on a licensed base, customisation and integration tasks are well-scoped. A 16-week delivery timeline is realistic with a 2-3 developer team. Schedule feasibility is MEDIUM–HIGH.

Legal Feasibility

The platform collects personal data and processes payments. GDPR compliance, privacy policy, terms of service, and a data processing agreement are required. The SOSANA KYC integration uses on-chain data, reducing PII storage on the platform. Legal feasibility is MEDIUM (pending DPA completion).

3.4 Process Model

TrustBOB was developed using an Agile (Scrum-inspired) incremental development model. SOSANA-specific customisations applied as feature branches.

Why Agile? Requirements evolved alongside SOSANA's blockchain ecosystem. The complaint module (with on-chain eligibility checks) was added in a later iteration, making Agile's iterative approach ideal for accommodating changing business requirements without disrupting the stable core.

4.1 Use Case Scenarios

Scenario UC-05: File a Complaint

Use Case IDUC-05
Use Case NameFile a Complaint Against a Business
ActorRegistered User (SOSANA KYC Verified)
PreconditionsUser is logged in, email verified, SOSANA KYC passed, token holding requirement met.
Main Flow 1. User navigates to business profile page.
2. Clicks "File a Complaint".
3. System calls SOSANA dApp API to verify KYC and token eligibility.
4. User selects complaint category (e.g., Rug Pull, Exit Scam).
5. User fills in title, description, and uploads evidence (attachments).
6. User submits the complaint form.
7. System stores complaint with status = FILED and sets response deadline.
8. Email notification sent to business owner and user.
9. Complaint is publicly visible on the business profile.
Alternate Flow3a. If KYC not verified → system shows error with link to SOSANA dApp.
3b. If token holding insufficient → error with eligibility link.
PostconditionsComplaint stored in DB, notifications sent, business owner can respond.

Scenario UC-04: Write a Review

Use Case IDUC-04
Use Case NameWrite a Review for a Business
ActorRegistered User (optional: AI Addon enabled)
PreconditionsUser is logged in and on the business profile page.
Main Flow 1. User clicks "Write a Review".
2. User selects star rating (1–5) and optionally uses AI Review Writer.
3. User fills in review title and body, optionally uploads attachments.
4. User submits the form (Captcha verified if enabled).
5. Review stored with status PENDING (or PUBLISHED if moderation off).
6. Business rating statistics updated automatically.
Alternate FlowIf KYC required setting is ON → user must complete KYC first.
PostconditionsReview saved, business rating recalculated, email notification sent.
